# takedown.techcloudup.com — Domain Takedown Investigator

## Goal

Provide an on-demand investigation tool for suspected phishing or brand-impersonation domains.

- Enter a domain in the browser → live terminal streams the full investigation in real time
- Collects DNS records, WHOIS, SSL certificate, HTTP headers, robots.txt, and sitemap
- Outputs a structured summary with all data needed to file abuse reports

---

## Overview

| Item | Detail |
|------|--------|
| Role | Passive domain investigation and takedown evidence collection |
| Trigger | On-demand (user submits domain via web form) |
| Output | Real-time terminal stream + full investigation summary |
| Complements | WAF (blocks inbound attacks) + IDS (detects intrusions) — Takedown removes external threats |

---

## Tech Stack

| Layer | Technology | Notes |
|-------|------------|-------|
| Domain / DNS | `takedown.techcloudup.com` | Cloudflare DNS (proxy ON) — same IP as WAF VM |
| Investigation Engine | Bash script (`investigate.sh`) | Runs on WAF VM — passive queries only (DNS, WHOIS, HTTP GET) |
| Backend API | Python Flask (extended from WAF/IDS app) | `GET /api/investigate/stream` — SSE endpoint, streams script output |
| Streaming | Server-Sent Events (SSE) | Line-by-line output pushed to browser as script runs |
| Frontend | Static HTML / CSS / JS | Served by Nginx — terminal-style UI with progress bar |
| Rate Limiting | In-memory lock (90s cooldown) | Prevents concurrent investigations on e2-micro VM |

## Investigation Steps (11 total)

| Step | Output File | Content |
|------|-------------|---------|
| 1 | `02_dig_A.txt` | A record (IP addresses) |
| 2 | `03_dig_NS.txt` | Nameservers |
| 3 | `04_dig_records.txt` | A, AAAA, MX, TXT, SOA, CNAME records |
| 4 | `05_ip_whois.txt` | Domain WHOIS + IP WHOIS |
| 5 | `06_traceroute.txt` | Network path (max 15 hops, 30s timeout) |
| 6 | `07_ssl_cert.txt` | SSL certificate chain + parsed details |
| 7 | `08_curl_headers.txt` | HTTP/HTTPS response headers (follows redirects) |
| 8 | `09_curl_response_headers.txt` | Verbose headers including CF-RAY, server type |
| 9 | `10_source_code.html` | Full HTML source of homepage |
| 10 | `11_sitemap.txt` | robots.txt + sitemap XML + extracted URLs |
| 11 | `12_summary.txt` | Consolidated summary with reporting channels |

All output files are created in a temporary directory and deleted after the stream completes.

## Security Design

| Item | Measure |
|------|---------|
| Domain input validation | Strict regex before any shell execution |
| Rate limiting | 90-second cooldown between investigations (in-memory lock) |
| No persistent storage | Investigation files written to `tmpdir`, deleted on completion |
| Passive queries only | No active scanning — DNS, WHOIS, HTTP GET, SSL handshake only |
| Cloudflare detection | 3-step check: NS pattern → IP range function → CF-RAY header |

---

## Data Flow

```
1. User     →  enters domain in browser form
2. Browser  →  GET /api/investigate/stream?domain=<domain>  (SSE connection)
3. Flask    →  validates domain regex
              → checks 90s rate limit
              → spawns investigate.sh in tmpdir (subprocess)
              → streams each output line as SSE event
4. Browser  →  renders lines in terminal UI with color coding + progress bar
5. Flask    →  on completion: cleans up tmpdir, sends __DONE__ event
6. Browser  →  shows "Investigation complete" badge
```
